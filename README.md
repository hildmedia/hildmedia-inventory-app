# Hildmedia Inventory App

Ein cleanes MVP für Kamera-Equipment-Inventar mit React, TypeScript, Vite, Supabase und Barcode-Scanning über `html5-qrcode`.

## 1. Installation

```bash
npm install
```

## 2. Supabase Tabelle anlegen

Öffne dein Supabase Projekt, gehe zu **SQL Editor** und führe dieses SQL aus:

```sql
create table if not exists workspaces (
  id uuid primary key default gen_random_uuid(),
  name text not null,
  created_at timestamptz not null default now()
);

insert into workspaces (id, name)
values ('00000000-0000-0000-0000-000000000001', 'HILD MEDIA')
on conflict (id) do nothing;

create table if not exists equipment (
  id bigint generated by default as identity primary key,
  workspace_id uuid not null references workspaces(id) default '00000000-0000-0000-0000-000000000001',
  name text not null,
  brand text,
  model text,
  serial_number text,
  ean_code text,
  category text not null check (
    category in (
      'Kamera',
      'Audio',
      'Licht',
      'Grip & Stative',
      'Computer und Pulte',
      'Signaltechnik & Wandlung',
      'Akkus',
      'Kabel',
      'Speicher & Medien',
      'Zubehör',
      'Werkzeug & Verbrauchsmaterial',
      'Cases & Taschen'
    )
  ),
  purchase_date date,
  purchase_price numeric(12, 2),
  status text not null default 'verfügbar' check (
    status in ('verfügbar', 'verliehen', 'in Reparatur', 'ausgemustert')
  ),
  borrower text,
  borrowed_at date,
  return_date date,
  notes text,
  created_at timestamptz not null default now(),
  updated_at timestamptz not null default now()
);
```

Damit `updated_at` bei späteren Änderungen automatisch aktualisiert wird:

```sql
create or replace function set_updated_at()
returns trigger as $$
begin
  new.updated_at = now();
  return new;
end;
$$ language plpgsql;

drop trigger if exists set_equipment_updated_at on equipment;

create trigger set_equipment_updated_at
before update on equipment
for each row
execute function set_updated_at();
```

Wenn deine Tabelle schon existiert, führe zusätzlich aus:

```sql
alter table equipment
add column if not exists updated_at timestamptz not null default now();
```

Für die Dokumentation von Check-Out und Check-In wird zusätzlich diese Tabelle genutzt:

```sql
create table if not exists customers (
  id bigint generated by default as identity primary key,
  workspace_id uuid not null references workspaces(id) default '00000000-0000-0000-0000-000000000001',
  customer_type text not null check (
    customer_type in ('Privatperson', 'Firma', 'Selbständig/Freiberuflich')
  ),
  name text not null,
  contact_person text,
  email text,
  phone text,
  notes text,
  created_at timestamptz not null default now(),
  updated_at timestamptz not null default now()
);
```

```sql
create table if not exists equipment_checkout_logs (
  id bigint generated by default as identity primary key,
  workspace_id uuid not null references workspaces(id) default '00000000-0000-0000-0000-000000000001',
  equipment_id bigint references equipment(id) on delete set null,
  customer_id bigint references customers(id) on delete set null,
  equipment_name text not null,
  action_type text not null check (action_type in ('Verleih', 'Reparatur')),
  checked_out_at timestamptz not null default now(),
  borrower text,
  borrowed_at date,
  return_date date,
  loan_notes text,
  repair_sent_at date,
  repair_notes text,
  checked_in_at timestamptz,
  created_at timestamptz not null default now()
);
```

Wenn die Tabelle schon existiert, ergänze die neue Spalte so:

```sql
alter table equipment_checkout_logs
add column if not exists checked_out_at timestamptz not null default now();

alter table equipment_checkout_logs
add column if not exists loan_notes text;

alter table equipment_checkout_logs
add column if not exists customer_id bigint references customers(id) on delete set null;
```

Wenn deine `customers` Tabelle schon existiert, passe den Check-Constraint für den neuen Typ in Supabase an:

```sql
alter table customers drop constraint if exists customers_customer_type_check;

alter table customers
add constraint customers_customer_type_check
check (customer_type in ('Privatperson', 'Firma', 'Selbständig/Freiberuflich'));
```

Für dieses MVP ohne Login muss Row Level Security entweder ausgeschaltet bleiben, oder du brauchst einfache Policies. Für einen privaten Test ist der einfachste Start:

```sql
alter table equipment disable row level security;
```

```env
VITE_DEFAULT_WORKSPACE_ID=00000000-0000-0000-0000-000000000001
```

## 4. Login aktivieren

In Supabase:

1. Gehe zu **Authentication > Users**.
2. Klicke **Add user**.
3. Lege deine E-Mail und ein Passwort an.
4. Setze bei Bedarf **Auto Confirm User**, damit du dich direkt einloggen kannst.

Danach Row Level Security aktivieren:

```sql
alter table equipment enable row level security;
alter table customers enable row level security;
alter table equipment_checkout_logs enable row level security;
alter table workspaces enable row level security;

drop policy if exists "authenticated can manage equipment" on equipment;
drop policy if exists "authenticated can manage customers" on customers;
drop policy if exists "authenticated can manage checkout logs" on equipment_checkout_logs;
drop policy if exists "authenticated can read workspaces" on workspaces;

create policy "authenticated can manage equipment"
on equipment
for all
to authenticated
using (workspace_id = '00000000-0000-0000-0000-000000000001')
with check (workspace_id = '00000000-0000-0000-0000-000000000001');

create policy "authenticated can manage customers"
on customers
for all
to authenticated
using (workspace_id = '00000000-0000-0000-0000-000000000001')
with check (workspace_id = '00000000-0000-0000-0000-000000000001');

create policy "authenticated can manage checkout logs"
on equipment_checkout_logs
for all
to authenticated
using (workspace_id = '00000000-0000-0000-0000-000000000001')
with check (workspace_id = '00000000-0000-0000-0000-000000000001');

create policy "authenticated can read workspaces"
on workspaces
for select
to authenticated
using (id = '00000000-0000-0000-0000-000000000001');
```

## 7. Hosting und iPhone PWA

Die App ist als PWA vorbereitet mit:

- `manifest.webmanifest`
- App Icons in `public/icons`
- iPhone Homescreen Meta-Tags
- Service Worker Registrierung

Beim Hosting, zum Beispiel Vercel oder Netlify, diese Environment Variablen setzen:

```env
VITE_SUPABASE_URL=https://dein-projekt.supabase.co
VITE_SUPABASE_ANON_KEY=dein-anon-key
VITE_DEFAULT_WORKSPACE_ID=00000000-0000-0000-0000-000000000001
```

Wichtig fuer den Scanner: Die gehostete App muss ueber HTTPS laufen.

Auf dem iPhone:

1. Safari oeffnen.
2. Gehostete App-URL aufrufen.
3. Teilen-Button tippen.
4. **Zum Home-Bildschirm** auswaehlen.
5. App vom Homescreen starten.

## Was das MVP kann

- Equipment hinzufügen
- Alle Equipment-Einträge aus der Supabase Tabelle `equipment` laden
- Suche nach Name, Marke, Modell, Seriennummer, EAN, Ausleiher und Notizen
- Filter nach Kategorie
- Filter nach Status
- Gesamtwert der aktuell gefilterten Inventarliste anzeigen
- EAN/Barcode per Kamera scannen und ins Formular übernehmen
- Erstell- und Änderungsdatum anzeigen
